1. SwiftData（若分支與 SwiftData 有關）
   - 是否有 import SwiftData、使用 @Model struct/enum。
   - 是否有 @Query 來查詢資料而非手動 fetch。
   - 是否使用 Environment modelContext / ModelContainer。
   - 是否有 conflict resolution 或 cloud sync 設定（若同步到 iCloud）。

2. Swift Concurrency（必檢）
   - 是否使用 async/await、Task、TaskGroup、actor。
   - 是否在需要的地方標註 @MainActor。
   - 有無避免直接在主執行序做同步 I/O。

3. Observation / 新觀測 API
   - 是否使用新的 Observation 宏/協議（例如 @Observable 或 Observation framework）。
   - 是否把模型層與 View 層分離且用觀測屬性驅動 UI。

4. SwiftUI 最佳實踐
   - 使用 NavigationStack / NavigationSplitView（取代老舊 NavigationView）。
   - 使用 enum 管理複雜 presentation state（sheet、fullScreenCover 等）。
   - 避免過度傳遞 Binding，盡量使用 Environment 或單一來源狀態。

5. SPM（模組化 / 相依管理）
   - 有無 Package.swift、是否把共用功能拆成 Swift packages（例如 SpecKit）。
   - 檢查第三方套件（如 SwiftUINavigation、SwiftDate）是否透過 SPM 加入。

6. iCloud / CloudKit 同步
   - 檢查是否設定 iCloud 容器（Xcode project entitlements）與 CloudKit 使用。
   - 若使用 SwiftData 的雲端功能，檢查相關 config。

7. 通知 / App Intents
   - 檢查是否使用新的通知 API、是否有權限處理與回呼流程、是否支援 App Intents（如有需要）。

8. AI / Foundation Models / ML
   - 檢查是否使用 Apple 的 FoundationModels、Core ML / CreateML、或呼叫外部 LLM API。
   - 檢查資料隱私、輸入輸出處理、批次與錯誤處理（非同步）。

9. 測試、CI 與可維運性
   - 是否有 Unit tests / UI tests、是否有 github actions 或 CI 設定。
   - 是否使用 SwiftLint /格式化工具（良好工程實務）。
